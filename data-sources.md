# Data Sources

## Governance and trust model

Each rate, fee, and policy field is treated as time-sensitive data with:

- Effective date
- Source reference
- Last verified timestamp
- Status flag (verified, estimated, promotional, expired)

## Source hierarchy

1. Official institution pricing pages and fee guides
2. Official policy and regulatory publications
3. Official product terms and conditions
4. Secondary reporting (used only with clear source attribution)

## Update cadence

- Daily where frequent rate movement is expected
- Weekly for active comparison categories
- Monthly for policy or lower-volatility pages

## Audit trail requirements

- Retain prior values for historical traceability
- Record update actor and timestamp
- Preserve source links for each material field

## User visibility

Every comparison output should show:

- Last updated time
- Verification status
- Source reference path

## Caution

Rates and terms can change between updates. Users should verify critical values directly with providers before acting.
